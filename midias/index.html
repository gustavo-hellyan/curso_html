Imagens:

- A imagem precisa tá na mesma pasta do código
- Em source, voce digita o nome da imagem e formato
- voce pode apertar control espaço para o Visual studio Code exibir as imagens que estao na pasta do código
- em alt (de alternative), voce bota o texto que explica a imagem (usado para leitores de tela - acessibilidade)
- Você também pode pegar imagens da internet. Basta colocar o endereço (link) dela

Imagens dinâmicas

Se o navegador tiver até 700 pixels, vai carregar a iamgem A. Se tiver até 1000 pixels, vai carregar a imagem B. Se não for nenhum dos casos, vai carregar a imagem C. A ordem de comandos é relevante.
Isso é importante, pois vai melhorar a experiência do usuário. E quanto mais o usuário permanecer no site, mas o Google classifica o seu site como de boa experiência e recomenda para outras pessoas - ao pesquisarem um conteúdo do seu site.

Audio

Podemos colocar audios mp3, ogg ou wav (esse é muito pesado, então não use em sites):

Atributo controls mostra o player de audio; atributo autoplay faz o audio rodar assim que carregar a pagina; loop faz o audio se repetir
preload="auto" ou deixar vazio: ao carregar o site, ele carrega automaticamente todos os dados e o arquivo, porém isso é ruim quando o arquivo é muito grande. Nesse caso, o site só terminaria de carregar quando todos os dados forem carregados.
preload="metadata" ele carrega somente os metadados e só carrega o arquivo quando o usuário clicar
preload="none" ele não carrega nada

Nesse modo abaixo, ele vai tentar carregar o primeiro arquivo, se nao conseguir, vai carregar o segundo arquivo
